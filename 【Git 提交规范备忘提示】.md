 【Git 提交规范备忘提示】
 -----------------------------------------------------------------
  - type (必填):
    - feat     : 新增功能          fix      : 修复 Bug
    - docs     : 修改文档          style    : 格式调整 (不影响逻辑)
    - refactor : 代码重构          perf     : 性能优化
    - test     : 测试用例          build    : 构建系统/依赖变更
    - ci       : 持续集成配置       chore    : 杂项事务
    - revert   : 撤销提交

  - scope (选填): 修改的影响范围 (如: router, auth, payment)。
  - subject (必填): 简要说明，首字母小写，结尾不加句号。

  - body (选填): 详细描述本次修改的动机，以及与之前逻辑的对比。
  - footer (选填): 关联并关闭 Issue (如: Closes #123) 或破坏性变更说明。
-----------------------------------------------------------------
